# Automatic CME Detection and Geometric Analysis

This repository provides post-training inference and downstream analysis code for coronal mass ejection (CME) region extraction and geometric measurement from coronagraph images. The full MAE pretraining pipeline is not included.

The workflow is organized into two stages:

1. **Segmentation stage** (`inference.py`): generates the CME mask and visualization outputs.
2. **Geometric analysis stage** (`rectangular_analysis.py`): computes angular and radial CME parameters from the segmented mask.

This repository is designed to run with a **trained model checkpoint** and does **not** include the full pretraining pipeline.

## Features

- Vision Transformer feature extraction
- PCA-based heatmap generation
- GraphCut-based CME segmentation
- Intensity-guided post-processing
- Rectangular-coordinate visualization of the segmented CME region
- CME geometric analysis, including:
  - start angle (`Tstart`)
  - end angle (`Tend`)
  - angular width (`AW`)
  - central position angle (`CPA`)
  - official clockwise CPA (`CPA_official`)
  - maximum height (`Rmax`)

## Scope

This repository provides **post-training inference and downstream analysis code only**. It is intended to be used with a trained model checkpoint and does not include the full training or pretraining pipeline.

## Input

### Input for `inference.py`

The segmentation stage expects:

- a trained model checkpoint
- a preprocessed coronagraph image
- image size of `512 × 512`

The current implementation is designed for running-difference coronagraph images in PNG format.

### Input for `rectangular_analysis.py`

The geometric analysis stage expects:

- the segmentation overlay image generated from the previous stage
- specifically, an image in which the CME mask is shown in green on top of the grayscale coronagraph background

In the current workflow, the default input for this stage is the `gray_with_mask_overlay.png` file generated by `inference.py`.

## Usage

### Step 1: CME segmentation

Run the segmentation script with:

```bash
python inference.py \
  --model_path checkpoints/checkpoint.pth \
  --image_path examples/sample_input.png \
  --output_dir outputs/sample_run \
  --pca_thresh 200 \
  --gray_img_thresh 190 \
  --brightness_base 40 \
  --close_to_base 10
```

### Step 2: Rectangular-coordinate analysis

Run the downstream analysis script with:

```bash
python rectangular_analysis.py \
  --input_path outputs/sample_run/gray_with_mask_overlay.png \
  --output_dir outputs/sample_run/rectangular_analysis
```

## Arguments

### `inference.py`

- `--model_path`: path to the model checkpoint
- `--image_path`: path to the input image
- `--output_dir`: directory to save the outputs
- `--img_size`: input image size, default `512`
- `--patch_size`: patch size, default `16`
- `--pca_thresh`: threshold for selecting high-confidence PCA regions
- `--gray_img_thresh`: threshold for removing low-intensity pixels from the mask
- `--brightness_base`: brightness margin for removing weak false-positive regions
- `--close_to_base`: brightness tolerance for merging nearby pixels
- `--expand_tolerance`: brightness tolerance used in boundary expansion
- `--merge_kernel_size`: neighborhood size used for main-region merging
- `--min_area`: minimum connected-component area used in mask filtering inside the segmentation script

### `rectangular_analysis.py`

- `--input_path`: path to the green-mask overlay image
- `--output_dir`: directory to save the analysis outputs
- `--solar_radius_pixels`: assumed solar radius in pixels, default depends on the script setting
- `--min_size`: minimum connected-component size for removing isolated noise regions
- `--dpi`: output figure resolution

## Output

### Output of `inference.py`

The segmentation stage saves the following files in `output_dir`:

- `pca_heatmap.png`: PCA-based saliency heatmap
- `refined_on_image.png`: segmentation mask rendered in green
- `gray_input.png`: grayscale version of the input image
- `gray_with_contour.png`: grayscale image with the CME contour overlay
- `gray_with_mask_overlay.png`: grayscale image with the full CME mask overlay
- `CME_region_analysis.txt`: basic region summary produced by the segmentation stage

### Output of `rectangular_analysis.py`

The downstream analysis stage saves:

- `polar_to_rectangular.png`: rectangular-coordinate plot of the segmented CME region
- `cme_angle_summary.txt`: final angular and radial CME summary

The summary file contains:

- `Tstart` (Start Angle)
- `Tend` (End Angle)
- `AW` (Angular Width)
- `CPA` (Central Position Angle)
- `Rmax` (Maximum Height)

## Notes

- This repository currently provides **post-training inference and downstream analysis code only**.
- The segmentation and the final geometric measurement are performed in **two separate scripts**.
- `inference.py` does **not** directly produce the final rectangular-coordinate angle summary.
- The final geometric parameters are computed by `rectangular_analysis.py` from the segmented green-mask image.
- The Vision Transformer backbone is used for feature extraction.
- The final segmentation result depends on both the learned representation and the post-processing parameters.

## Training Note

The released code in this repository focuses on **inference and downstream geometric analysis after model training**.
